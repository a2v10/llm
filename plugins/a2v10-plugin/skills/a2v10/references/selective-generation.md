# Выборочная генерация (блоки)

По умолчанию (если не указано иное) — генерируй все файлы эндпоинта.

Пользователь может указать конкретные файлы или блоки. Распознавай по ключевым словам.

> Расширение XAML-файлов ниже (`.xaml`) — для краткости. На диске оно берётся из `XAML naming convention` проекта (`.vxaml` или `.xaml`, CLAUDE.md); создавая файл — ставь его.

## Файлы целиком

| Что сказал пользователь | Что генерировать |
|---|---|
| `logic.sql`, "все процедуры", "хранимки" | весь `logic.sql` |
| `schema.sql`, "схема", "таблица" | `schema.sql` |
| `model.json` | `model.json` |
| `index.view.xaml`, "xaml индекса", "страница списка" | `index.view.xaml` |
| `edit.dialog.xaml`, "форма редактирования", "диалог редактирования" | `edit.dialog.xaml` |
| `browse.dialog.xaml`, "диалог выбора", "browse" | `browse.dialog.xaml` |
| `index.template.ts`, `edit.template.ts` | соответствующий template |
| `index.d.ts`, `edit.d.ts` | соответствующий d.ts |

## Отдельные процедуры

| Что сказал пользователь | Процедура |
|---|---|
| "процедура Index", "процедура списка" | `<Entity>.Index` |
| "процедура Load" | `<Entity>.Load` |
| "процедура Metadata" / "процедура Update" | `<Entity>.Metadata` + `TableType` + `<Entity>.Update` (всегда вместе) |
| "процедура Fetch" | `<Entity>.Fetch` |
| "процедура Delete" | `<Entity>.Delete` |

## Формат вывода в чате

Каждый файл — отдельный код-блок с именем файла в заголовке:

````
**logic.sql** (процедура Index)
```sql
...
```
````

Если запрошено несколько блоков — выводи последовательно.
